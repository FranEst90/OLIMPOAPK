Sección 1

¿Qué es OLIMPO?

OLIMPO es una interfaz cliente independiente que expone servicios de ALFA-1 sin modificar el bot original. Los módulos TempMail y SMSPool ya existen y funcionan — OLIMPO los conecta a una UI propia con acceso controlado.

Principios de diseño
Independencia total de ALFA-1 — OLIMPO no modifica ni depende del código del bot. Consume las mismas APIs externas directamente.
Un bot dedicado para auth — Bot de Telegram exclusivo para enviar códigos OTP. No comparte instancia con ALFA-1.
Streamlit como backend UI — Railway hostea el servidor Streamlit. El APK es solo un contenedor WebView sin lógica propia.
Whitelist por Telegram ID — Solo IDs autorizados explícitamente pueden acceder. Sin registro público.

Sección 3

Sistema de autenticación

Login con Telegram ID + OTP enviado por el bot dedicado de OLIMPO. Sin contraseñas. Sin registro público.

Flujo completo de login
1
Usuario abre APK → WebView carga https://olimpo.railway.app
2
Streamlit muestra campo: "Ingresa tu Telegram ID"
3
Streamlit verifica ID en WHITELIST → si no está: acceso denegado
4
Genera OTP de 6 dígitos → bot OLIMPO lo envía al Telegram del usuario
5
Usuario ingresa OTP en Streamlit → si coincide: sesión iniciada (60 min)
6
OTP expira en 5 minutos. Máximo 3 intentos por sesión.
auth.py — estructura del módulo Nuevo archivo
import secrets, time from telegram import Bot # OTPs activos: {tg_id: {code, expires_at, attempts}} _pending: dict = {} async def send_otp(tg_id: int, bot: Bot) -> str: code = str(secrets.randbelow(900000) + 100000) # 6 dígitos _pending[tg_id] = { "code": code, "expires_at": time.time() + 300, # 5 minutos "attempts": 0, } await bot.send_message( chat_id=tg_id, text=f"🔐 Tu código OLIMPO: {code}\nExpira en 5 minutos.", parse_mode="HTML" ) return code def verify_otp(tg_id: int, entered: str) -> bool: entry = _pending.get(tg_id) if not entry: return False entry["attempts"] += 1 if entry["attempts"] > 3: _pending.pop(tg_id, None) return False if time.time() > entry["expires_at"]: return False ok = entered.strip() == entry["code"] if ok: _pending.pop(tg_id, None) return ok def is_whitelisted(tg_id: int) -> bool: import os ids = [x.strip() for x in os.getenv("OLIMPO_WHITELIST", "").split(",")] return str(tg_id) in ids
Sección 4 · Módulo

TempMail — mail.tm

Correos temporales via api.mail.tm. Cada usuario tiene una cuenta activa con token JWT que se renueva automáticamente cada 55 minutos. También genera identidades mexicanas completas (CURP, NSS, datos personales).

API base: https://api.mail.tm REST · Bearer Token
Método HTTP Endpoint Acción Auth
GET /domains Listar dominios disponibles para crear cuenta —
POST /accounts Crear cuenta de correo temporal —
POST /token Obtener JWT para sesión —
GET /messages Listar mensajes en bandeja Bearer
GET /messages/{id} Leer mensaje completo (subject + body) Bearer
DELETE /messages/{id} Eliminar mensaje Bearer
DELETE /accounts/{id} Eliminar cuenta completa Bearer
Funciones a construir en OLIMPO
FunciónRetornaDescripción
crear_cuenta(user_id) dict Crea cuenta en mail.tm, guarda email + password en SQLite
Usa dominio aleatorio de GET /domains
get_token(user_id) str Devuelve JWT activo. Si tiene más de 55 min, renueva automáticamente
ver_bandeja(user_id) list[dict] Lista hasta 10 mensajes con subject, remitente y estado leído/no leído
leer_mensaje(user_id, msg_id) dict Cuerpo completo del mensaje. Trunca a 3000 chars si es muy largo
eliminar_mensaje(user_id, msg_id) bool Elimina mensaje de mail.tm
generar_identidad(user_id) dict Genera nombre, CURP, NSS, teléfono, dirección MX ficticios con Faker
eliminar_cuenta(user_id) None Borra cuenta en mail.tm y limpia registro en SQLite

SMS Pool — smspool.net

Números virtuales para recibir SMS de verificación. La API usa SMSPOOL_API_KEY en cada request. El polling busca el código cada 15 segundos hasta recibir status 3 (completado).

API base: https://api.smspool.net REST · API Key en body
EndpointParams claveRespuesta importante
GET /country/retrieve_all — Lista de países con ID y nombre
GET /service/retrieve_all — Lista de servicios (WhatsApp, Google, etc.)
POST /purchase/sms country, service, apikey order_id, number, expires_in
POST /sms/check orderid, apikey status (1=pendiente, 3=recibido), sms (el código)
POST /sms/cancel orderid, apikey Cancela pedido y devuelve créditos en smspool
POST /request/balance apikey Saldo USD disponible en la cuenta smspool
Status codes de /sms/check
StatusSignificadoAcción
1Pendiente — SMS no ha llegadoSeguir polleando
3Completado — SMS recibidoLeer campo sms (NO code)
otroError / expiradoMostrar error, no reintentar
⚠ CRÍTICO

El código SMS llega en el campo sms de la respuesta, nunca en code. El campo code no existe en esta API.

Identidad visual OLIMPO

Toda la navegación e interfaz debe seguir esta paleta. El banner define el tono: oscuro absoluto, rojo/naranja con brillo, tipografía monoespaciada estilo militar-retro.

Paleta de colores oficial

void black
#1C0800
ember deep
#6B1800
crimson border
#D42000
olimpo red
#FF6030
ember orange
#D4B89A
parchment text
Reglas de interfaz
Fondo siempre oscuro — #080808 como base. Nunca fondos blancos ni claros en ninguna pantalla.
Texto principal — #D4B89A (parchment). Nunca blanco puro — el contraste con el negro es demasiado agresivo.
Acciones primarias — botones con borde #D42000, fondo semi-transparente del mismo rojo. Texto #FF6030.
Tipografía display — monoespaciada, mayúsculas, letter-spacing amplio. Para títulos y el nombre OLIMPO.
Divisores — línea 1px solid #6B1800. Nunca gris neutro.
Códigos y datos — siempre en font-family: monospace, color #FF6030.
Éxito — verde oscuro #2A6A30. Error — rojo #D42000. Nunca verde brillante ni rojo saturado.
Sin bordes redondeados excesivos — máximo border-radius: 6px. Estética angular, no bubble.
Streamlit — configuración de tema
# .streamlit/config.toml [theme] base = "dark" backgroundColor = "#080808" secondaryBackgroundColor = "#1C0800" primaryColor = "#D42000" textColor = "#D4B89A" font = "monospace"

APK Android — WebView

Contenedor mínimo en Kotlin. No tiene lógica propia — solo carga la URL de Railway en un WebView. GitHub Actions genera el APK firmado en cada push a main.

MainActivity.kt — código completo ~30 líneas
class MainActivity : AppCompatActivity() { override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) val webView = WebView(this) setContentView(webView) webView.settings.apply { javaScriptEnabled = true domStorageEnabled = true loadWithOverviewMode = true useWideViewPort = true } webView.webViewClient = WebViewClient() webView.loadUrl("https://olimpo.up.railway.app") } // Manejar botón atrás dentro del WebView override fun onBackPressed() { if (webView.canGoBack()) webView.goBack() else super.onBackPressed() } }
.github/workflows/build-apk.yml GitHub Actions
name: Build APK on: push: branches: [main] jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-java@v4 with: java-version: '17' distribution: 'temurin' - name: Build APK run: ./gradlew assembleRelease - name: Sign APK uses: r0adkll/sign-android-release@v1 with: releaseDirectory: app/build/outputs/apk/release signingKeyBase64: ${{ secrets.KEYSTORE_BASE64 }} alias: ${{ secrets.KEY_ALIAS }} keyStorePassword: ${{ secrets.KEYSTORE_PASSWORD }} keyPassword: ${{ secrets.KEY_PASSWORD }} - uses: actions/upload-artifact@v4 with: name: olimpo-release.apk path: app/build/outputs/apk/release/*.apk
Secrets requeridos en el repo GitHub
KEYSTORE_BASE64 — keystore en base64. Generar con: keytool -genkey -v -keystore olimpo.jks
KEY_ALIAS — alias elegido al crear el keystore
KEYSTORE_PASSWORD — contraseña del keystore
KEY_PASSWORD — contraseña de la clave
